Desenvolvimento

Nesta página, encontra-se como se deu o desenvolvimento da "iEstufa".

Materiais e Métodos

Hardware

Sensor de Umidade e Temperatura DHT11: Inicialmente esse sensor cuidaria de ambos, temperatura e umidade, porém, confirmamos que, no próprio laboratório, já havia um sensor de umidade mais preciso, assim o sensor DHT11 foi usado apenas para medir a temperatura da iEstufa.

Sensor de Umidade de Solo (YI-69): Esse sensor já estava disponível no laboratório antes da realização do trabalho e nos foi disponibilizado para a realização do mesmo. Optamos por usar esse sensor ao invés do DHT11 para medir a umidade do solo pois ele se mostrou mais preciso.

Sensor Ultrassônico HC-SR04: Esse sensor foi utilizado para medir o nível da água dentro do recipiente onde ela seria armazenada. No momento que a quantidade disponível de água se torna insuficiente, um LED presente na protoboard que está geralmente aceso, se apaga, notificando os usuários de que é necessário reabastecer o suprimento de água.

Placa Arduino UNO: É o microcontrolador utilizado para controlar e integrar os componentes do circuito.

Invólucro de Aquário: Recipiente utilizado para criar um ambiente onde a temperatura e umidade pudessem ser regulados.

Tampa de Madeira: Tampa feita sob medida para o invólucro de aquário para facilitar a implementação de componentes como a lâmpada incandescente e para dar uma superfície sólida para apoiar partes físicas da iEstufa, como a protoboard e a placa Arduino UNO. Foi feita com o auxílio do professor Alexandre de Desenho Técnico Auxiliado por Computador.

Bomba de Aquário: Usada para retirar a água do recipiente onde esta é armazenada quando isso se faz necessário.

Lâmpada Incandescente: Usada para aquecer o ambiente da iEstufa quando é detectada uma temperatura abaixo de 19ºC.

Relés: Usados para controlar a bomba de aquário e a lâmpada incandescente. Quando é detectada a condição de ativação de alguma delas, seu respectivo relé passará a permitir a passagem de corrente, assim ativando o componente necessário.

Balde Comum: Recipiente onde é armazenada a água a ser usada na iEstufa.

Termômetro infravermelho: Usado para medir a temperatura dos componentes e ver se estes estavam funcionando corretamente.

Software

O software utilizado na iEstufa foi desenvolvido em C++, na IDE Arduino.

//Inclui a biblioteca "Ultrasonic.h", para que o sensor ultrassônico seja utilizado
#include <Ultrasonic.h>
//Inclui a biblioteca "DHT.h", para que o sensor de umidade e temperatura seja utilizado
#include "DHT.h"
//O segmento de código a seguir define constantes que indicam os pinos dos Relés e sensores
#define pino_sinal_analogico A0
#define DHTPIN A1
#define releLampada 7
#define releBomba 8
#define pino_trigger 4
#define pino_echo 5
//Define o tipo de sensor DHT(Umidade e temperatura)
#define DHTTYPE DHT11
//Variável para ler os valores do sensor de umiade,
int umidade;
//Inicializa o sensor ultrassônico nos pinos echo e trigger
Ultrasonic ultrasonic(pino_trigger, pino_echo);
//Inicializa o sensor de umidade e temperatura
DHT dht(DHTPIN, DHTTYPE);
//2 é o pino do LED de aviso do baixo nível
void setup() {
  pinMode(2, OUTPUT);
  pinMode(releLampada, OUTPUT);
  pinMode(releBomba, OUTPUT);
  Serial.begin(9600);
  dht.begin();
}

void loop() {
  digitalWrite(releBomba, HIGH);
  //Declaração e criação das variáveis que serão utilizadas ao decorrer do programa
  float Nivel, valor, temperatura;
  long microsec = ultrasonic.timing();
  Nivel= ultrasonic.convert(microsec, Ultrasonic::CM);
  temperatura=  dht.readTemperature();
  umidade = analogRead(pino_sinal_analogico);
  Serial.print("NIVEL ");
  Serial.print(Nivel);
  Serial.print("\n");
  Serial.print("TEMPERATURA: ");
  Serial.print(temperatura);
  Serial.print("\n");
  Serial.print("UMIDADE: ");
  Serial.print(umidade);
  Serial.print("\n");
  //O segmento de código a seguir informa a necessidade de se colocar ou não colocar mais água no reservatório
    if(Nivel>15){
      digitalWrite(2, HIGH);
    }
    else{
      digitalWrite(2, LOW);
    }
  delay(5000);
  //O segmento de código a seguir é um controlador on-off para controlar a temperatura
  if(temperatura<19){
    digitalWrite(7, LOW);
  }
  else{
    digitalWrite(7, HIGH);
  }
  delay(5000);
  //O segmento de código a seguir é um controlador on-off para controlar a umidade
  if(umidade > 800 && umidade < 1024 && Nivel>15){
    digitalWrite(releBomba, LOW);
    delay(3000);
    digitalWrite(releBomba, HIGH);
  }
  else{
    digitalWrite(releBomba, HIGH);
  }
  delay(10000);
}

Processo de Desenvolvimento

Durante o primeiro dia do trabalho, discutimos como o projeto seria realizado e que estratégias iríamos usar para realizar cada parte da iEstufa. O primeiro assunto abordado foi a produção da tampa para o aquário. Assim, produzimos desenhos de cada peça que seria usada na fabricação da tampa e os apresentamos ao professor Alexandre que nos auxiliou na produção desta. Mais tarde seria feito um furo na parte de cima da tampa para a introdução da lâmpada incandescente que aqueceria o sistema em caso de queda na temperatura. Para evitar que houvessem erros no projeto final, após a finalização do recipiente que teria a iEstufa, testamos separadamente cada um dos componentes. Não houveram problemas significativos no funcionamento de nenhum componente específico no sistema, com exceção da lâmpada, que não acendeu nos testes iniciais. Quanto à lâmpada, primeiro pensamos que o problema fosse com a boquilha usada na conexão, chegamos inclusive, a comprar uma boquilha nova para que fosse substituída, contudo, isso se mostrou desnecessário, já que em testes subsequentes a lâmpada funcionou normalmente. Imaginamos que ela não funcionou como esperado da primeira vez por conta de mal contato com algum fio. Após esse pequeno contratempo, os testes seguintes seguiram tranquilamente. Um problema muito recorrente durante a execução do trabalho eram os erros na IDE, uma vez que essa às vezes não reconhecia o arduino, não carregava ou não tinha as bibliotecas necessárias para os componentes, nesse qual era necessário só instalá-los pela internet. Nos outros casos, era necessário tentar trocar a porta do arduino, ou então, trocar de computador. Outro contratempo sem contudo, relação com os componentes eletrônicos, foi a amostra de terra utilizada para a realização do projeto. Esta acabou tendo de ser coletada durante um dia chuvoso, assim, ela ainda se encontrava molhada ao ser colocada dentro da iEstufa, situação que acabou sendo agravada pelos testes realizados com a bomba de água. Além disso, o aquário era antigo, e logo, quando colocava água nele, o cheiro de peixe que havia nele ressurgia, o que causava um odor desagradável.
